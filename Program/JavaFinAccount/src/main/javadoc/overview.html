Financial Accounting System Overview

Project Description

The Financial Accounting System is designed to manage and analyze financial records. The application provides features such as:

  • Adding records: Users can enter details about expenses and income specifying the amount, category (e.g., food, entertainment, transportation), and date.
  • Viewing data: Users can review their financial records through a user-friendly interface.
  • Statistics: The system generates graphical statistics of expenses and income, helping users visualize their financial health.
  • Data storage: Automatically saves information to a text file, allowing for data persistence between sessions.
  • Control: Operates through a console-based user interface where users navigate and interact via menu selections and data inputs.

Architecture and Class Structure

The application is structured using the onion architecture to ensure robust scalability and maintainability. This architecture centers the application's domain logic, surrounded by layers handling application services, infrastructure, and the user interface. Below is a breakdown of each layer and its components:

  1. Domain Layer:
    This is the core of the application, containing essential entities such as:
    • Record
    • Category
    • User
    • DateAndTime
    Value objects like StatisticField are used for specific data operations, primarily statistical computations.
  2. Application Services Layer:
    Acts as a mediator between the domain and infrastructure layers, with key services including:
    • RecordService
    • UserService
    • CategoryService
    • StatisticsService
    These services manage data flow and enforce business rules by delegating computations and operations to the domain layer. This structured approach ensures that operations are processed efficiently, aligning with the core business logic encapsulated in the domain.
  3. Infrastructure Layer:
    Implements interfaces defined by the core for data persistence and operations, comprising:
    • InMemoryRecordRepository
    • InMemoryCategoryRepository
    • InMemoryUserRepository
    This layer also includes serialization components like RecordSerializer for data integrity and conversion. It handles detailed interactions with data, particularly managing file-based data storage and in-memory data repositories.
  4. User Interface Layer:
    The top layer that interfaces directly with users through a Text User Interface (TUI). TUI uses controllers from the Controllers package enhancing user interaction with data managed at the infrastructure level and enabling access to application services for data processing.
